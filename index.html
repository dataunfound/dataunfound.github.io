How HTTP Requests Actually Work

Date: 17 March 2026

When you type a URL into your browser, it sends an HTTP request to a server[cite: 299]. This is basically a message asking for a specific file or page[cite: 300]. The server processes this and sends back an HTTP response, which includes a status code (like 200 OK) and the content itself[cite: 301, 302]. This cycle happens dozens of times just to load a single modern webpage, as it fetches images, CSS, and scripts separately[cite: 304].

Why Database Normalisation Matters

Date: 10 March 2026

Normalisation is all about organising a database so each table focuses on one topic[cite: 273, 274]. By splitting data into multiple tables and linking them with keys, we avoid duplicating data and keep things consistent[cite: 274, 275]. However, I recently learned that in massive systems where read speed is critical, engineers sometimes intentionally denormalise tables to avoid slow join operations. It is a really interesting trade-off.
