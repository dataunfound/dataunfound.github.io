Timestamp: 17-MAR-2026 // HTTP_Request_Cycle

When you type a URL into a browser, it fires an HTTP request to a remote server. This payload asks for a specific file. The server processes this request and returns an HTTP response, complete with a status code (e.g., 200 OK) and the payload itself. In modern distributed architectures, this cycle happens dozens of times per page load, fetching scripts and assets concurrently.

Timestamp: 10-MAR-2026 // Database_Normalisation

Normalisation is crucial for eliminating data redundancy. By splitting data into multiple tables, we ensure transactional consistency. However, I recently learned that in massive systems requiring ultra-fast read speeds, engineers intentionally denormalise tables. Avoiding complex JOIN operations drastically reduces read latency, highlighting a fascinating trade-off between structural purity and raw performance.
